# Coding Guidelines

## Function responsibilities

Functions should have a clear and limited responsibility.

Business logic should be separated from persistence and external service access.

## Error handling

Functions that can fail must provide clear error handling.

Invalid input must be detected before executing business logic.

## Maintainability

Code should use descriptive names and avoid duplicated logic.

Changes should remain consistent with the existing project structure.

Comments generated during Pull Request reviews must describe a concrete and actionable problem.
